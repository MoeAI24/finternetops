Agent #1 • Canonical Reference Implementation
Treasury Reconciliation Agent (TRA)

Continuously reconciles value movement across accounts, rails, and systems (banks, custodians, processors, tokenized rails), producing provable, auditable truth for finance teams—without replacing human approvals or core systems.

Back to Training Back to Agent Library

Purpose

Prepare reconciled truth: matched movements, timing differences, and real mismatches—then package evidence so humans can sign off.

Operator-firstAudit-readyRead-only inputs

Hard boundaries (non-negotiable)

The TRA never moves funds, executes trades, modifies ledgers, approves accounting decisions, or provides legal/tax advice. It prepares evidence; humans own decisions.

No fund movementNo approvalsNo ledger writes

Core workflow

Ingest → Normalize → Match → Classify differences → Explain with evidence → Queue exceptions → Package audit artifacts.

Deterministic rerunsTraceabilityException-first

Human-in-the-loop

People approve tolerances, resolve exceptions, sign reconciliations, and escalate anomalies. The agent reduces manual comparison work—not responsibility.

Separation of dutiesApprovals preservedEscalations

Risk & failure modes

Late feeds flagged as timing differences. Schema drift handled via versioned adapters + alerts. Partial outages degrade gracefully. No silent failures.

No silent failureSchema drift alertsGraceful degradation

Compliance posture

Immutable logs (hash-linked), source traceability per line item, deterministic reruns, and preserved separation of duties. Built to support SOX-style controls without claiming certification.
