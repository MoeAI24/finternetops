Settlement Watchdog

Surfaces settlement breaks, mismatches, and late-stage exceptions across tokenized workflows.

Tier: Operator Human-in-the-loop required Back to Catalog

What this agent does

“Settlement Watchdog cuts the time it takes to see and understand breaks—so your ops team resolves faster with a clean evidence trail.”

Best for
  • Ops teams handling settlement exceptions
  • Brokers/clearing-aligned teams needing visibility
  • Any desk trying to reduce T+ friction and late breaks
Bad fit / walk-away
  • Replacing control functions
  • No access to settlement status feeds
  • Expecting it to “fix” breaks automatically

Inputs & outputs

Inputs
  • Settlement status events (custody/clearing)
  • Trade confirmations (sanitized)
  • Reference data (counterparty, instrument)
  • Exception queue exports
Outputs
  • Exception triage queue (ranked)
  • Break reason classification (explainable)
  • Daily reconciliation snapshot
  • Escalation-ready notes
Note: Implementation specifics (prompts/config/orchestration) are gated by partner tier and certification (see IP Notice).

KPIs this agent improves

  • Time-to-triage breaks
  • Break aging reduction
  • First-touch resolution rate
  • Ops hours saved (measured, not promised)

Human-in-the-loop boundaries

  • Does not change books/records.
  • Does not submit instructions.
  • Human approval required for escalation/communications.
Approved language: “flags”, “surfaces”, “summarizes”, “supports review”. Avoid “decides”, “executes”, “guarantees”.

Where it fits (pilot → production)

  • Pilot: daily break queue + top-10 root causes
  • Production: near-real-time break surfacing + escalation matrix
  • Audit: change log + exception evidence
